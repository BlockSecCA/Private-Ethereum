Install eth stats tool (optional)

Installing the network status monitoring tools is described here: 
https://github.com/ethereum/wiki/wiki/Network-Status

The network monitoring has to components:
  • Stats collector (aka eth-net-intelligence-api)
  • The Web site (aka eth-netstats)
For the purposes of the private chain and network the Web site has been published in AWS EC2, for public access.

Installing the stat collector involves doing the following, using any of the accounts in the OS (ubuntu assumed)

    • git clone https://github.com/cubedro/eth-net-intelligence-api
    • cd eth-net-intelligence-api
    • npm install
    • sudo npm install -g pm2
Note: the following may not be present and could need git may not be installed by default so add it 
  • Git   "sudo apt install git". 
  • Npm "sudo apt install npm"
  • Node "sudo apt install nodejs-legacy"
Once the installation is completed just edit the app.json file in the "eth-net-intelligence-api" folder to contain information on the node to be collected. Example for collecting "usera" and "userb", to be pushed to the AWS Web site ("kaka" is the secret work used to communicate with the site).

This file is collecting from "localhost" with the assumption that the nodes "usera" and "userb" are in the same OS image.

Note the address for the AWS Web site may change every time the instance is rebooted.

To launch the collection just do the following from inside ""eth-net-intelligence-api" folder:
owner@ubuntu:~/eth-net-intelligence-api$ pm2 start app.json
